# RotDash — Brand Guide (v0)

Temporary branding for the **MVP / indie hacker launch phase**.
This will evolve later if the project grows into a full SaaS.

---

# 1. Brand Name

**RotDash**

Meaning:

* **Rot** → projects slowly decay when ignored
* **Dash** → a dashboard that exposes the decay

Core idea:

> A brutally honest dashboard that shows which of your projects are alive — and which are rotting.

Tone:

* honest
* slightly sarcastic
* indie hacker energy
* productivity without corporate fluff

---

# 2. Positioning

RotDash is **not a project manager**.

It is a **project decay detector**.

What it solves:

Many developers have:

* dozens of unfinished side projects
* MVPs abandoned after a weekend
* repos forgotten for months

RotDash answers one question:

> Which of my projects are silently rotting?

Instead of:

---

# 5. Core Concepts

The entire brand revolves around **project decay**.

Signals used:

* **rotScore**
* **activityStatus**
* **task completion**

Activity scale:

```
Hot
Warm
Stale
Cold
Glacier
```

Meaning:

```
Hot → actively worked on
Warm → slowing down
Stale → neglected
Cold → almost dead
Glacier → forgotten
```

---

# 6. Visual Identity

## Primary Colors

Activity colors map directly to status.

```
Hot       → #22c55e   (green)
Warm      → #eab308   (yellow)
Stale     → #f97316   (orange)
Cold      → #ef4444   (red)
Glacier   → #64748b   (gray)
```

These colors should appear in:

* activity badges
* charts
* rot indicators
* dashboard highlights

---

## Neutral Colors

Dark developer-friendly palette.

```
Background        → #0f172a
Surface           → #111827
Card              → #1f2933
Border            → #374151
Primary text      → #e5e7eb
Secondary text    → #9ca3af
```

# 10. Product Philosophy

RotDash follows three principles:

**1. Brutal honesty**

Projects rot when ignored.
The dashboard should show this clearly.

**2. Minimal friction**

No complicated project management.

Just:

* projects
* tasks
* activity signals

**3. Developer-first**

Built for:

* indie hackers
* solo builders
* side project addicts

* Jira
* Linear
* Notion

```
project decay visibility
```
